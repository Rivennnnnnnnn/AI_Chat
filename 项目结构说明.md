# AI_Chat 项目结构说明

本项目是一个基于 Go 语言开发的后端服务，采用典型的 Go 项目布局，将业务逻辑、数据模型和通用工具库进行分离。

## 目录结构

```text
AI_Chat/
├── configs/            # 配置文件目录
│   └── mysql.yaml      # MySQL 数据库配置
├── docs/               # 项目文档
│   └── API_Doc.md      # 接口文档
├── internal/           # 内部业务逻辑（不对外暴露）
│   ├── app/            # 应用启动及生命周期管理
│   │   └── app.go      # 初始化各个组件并启动应用
│   ├── model/          # 核心数据模型（对应数据库表或业务实体）
│   │   └── user_base.go# 用户基础信息模型
│   └── repository/     # 数据访问层（负责与数据库交互）
│       └── user_base.go# 用户基础信息相关的增删改查
├── logs/               # 日志文件存储目录
│   └── app.log         # 应用运行日志
├── pkg/                # 公共工具类及第三方库封装
│   ├── db/             # 数据库连接初始化
│   │   └── mysql.go    # GORM 初始化与连接池配置
│   └── utils/          # 常用工具函数
│       ├── configs.go  # 配置解析工具
│       ├── logger.go   # 日志初始化（基于 Zap 或标准库）
│       └── snowflake.go# 雪花算法 ID 生成器
├── 表结构设计/         # 数据库设计文档
│   └── 用户基本信息表.md
├── main.go             # 程序主入口
├── go.mod              # Go 依赖管理文件
├── go.sum              # Go 依赖校验文件
├── UserBase_test.go    # 单元测试/集成测试文件
└── 项目结构说明.md     # 本文件（项目结构导航）
```

## 主要模块说明

- **`main.go`**: 程序的起点，主要负责调用 `internal/app` 进行初始化。
- **`internal/app`**: 相当于整个应用的“组装机”，在这里进行配置加载、数据库连接、Repository 实例化等。
- **`internal/model`**: 定义了系统中使用的各种结构体，通常是与数据库表结构一一对应的持久化对象。
- **`internal/repository`**: 包含了所有的数据库操作逻辑，解耦了业务逻辑与具体的数据存储实现。
- **`pkg/`**: 存放一些通用的、可以被多个项目复用的代码。比如数据库驱动的初始化、雪花算法生成器、日志工具等。
- **`configs/`**: 统一存放 YAML 或 JSON 格式的配置文件。
